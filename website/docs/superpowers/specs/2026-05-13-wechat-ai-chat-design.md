# 微信AI聊天小程序 —— 产品与技术方案

## 项目概述

**小程序名称**：Lookmore撸猫

一个微信小程序，接入 lazymanchat.com 的AI聊天能力，供10人以内小团队内部使用。支持从20+个AI模型中选择，覆盖日常问答、编程、翻译、写作等全场景需求。

**分发方式**：微信体验版，邀请制，无需正式上线审核。

---

## 用户与使用场景

- **用户**：团队内部成员，约10人
- **注册资质**：微信个人身份注册小程序
- **使用场景**：全场景（问答、编程、翻译、写作、画图等）
- **访问控制**：通过微信体验版邀请机制控制，不额外设登录

---

## 系统架构

```
微信小程序 → HTTPS → 云服务器 (Node.js) → HTTPS → lazymanchat.com 内部API
```

### 为什么需要后端代理

1. lazymanchat.com 没有公开API，需要后端中转
2. lazymanchat.com 需要登录态，后端统一维护一个共享登录会话，所有团队成员共用
3. 微信小程序有域名白名单限制，只能请求已备案的HTTPS域名
4. 小程序前端代码公开，敏感逻辑（登录凭据等）放后端更安全

### 登录态管理

- 后端服务器启动时，使用你的 lazymanchat.com 账号自动登录，获取并维护 Cookie/Token
- 所有团队成员的小程序请求共用这个登录态
- 后端定时检测登录态是否过期，过期后自动重新登录
- 团队成员无需各自登录 lazymanchat，完全无感

### 服务器规格

- **选择**：阿里云 ECS
- **配置**：2核2G，3M带宽
- **系统**：Linux (Ubuntu 22.04)
- **运行环境**：Node.js 18+
- **月费**：约50元

## 页面设计

共3个页面：

### 1. 对话列表页（入口）

- 顶部居中显示品牌名称 **"Lookmore撸猫"**（类似百度首页Logo风格，简洁醒目）
- 下方显示所有历史对话，每条显示标题、模型名、消息数、时间
- 标题自动取第一条消息的前几个字
- 点击进入对应聊天页继续对话
- 长按可删除对话
- 底部 "+新建对话" 按钮

### 2. 聊天页（核心）

- 中间：消息列表，用户消息靠右（绿色气泡），AI回复靠左（白色气泡）
- AI回复左下角标注模型名称
- 底部输入区域：
  - 输入框上方：一排可横向滑动的模型快捷标签，当前选中高亮
  - 标签最右侧有 "+更多" 进入模型选择页
  - 输入框左侧：上传按钮（支持图片和文件），点击调起微信选择图片/文件
  - 输入框 + 发送按钮

### 3. 模型选择页

- 顶部两个分类标签：基础模型 / 增强模型
- 下方为对应分类的模型列表，单选（Radio样式）
- 底部 "确认选择" 按钮，返回聊天页并更新快捷标签

## 后端API设计

### GET /api/models

返回所有可用模型的列表及分类信息。

### POST /api/chat

发送聊天消息，返回AI回复。支持纯文本和带图片/文件的消息。

请求：`{ model: "模型名", message: "用户消息", files?: [{ name, type, base64 }] }`
响应：`{ reply: "AI回复内容" }`

- 图片/文件由小程序端转为 base64 后上传
- MVP阶段使用一次性返回（非流式），后续可升级为流式输出

---

## 数据存储

- 对话数据存储在服务器本地，使用 JSON 文件或 SQLite
- 每个对话保存：标题、消息列表（每条含角色、内容、模型名、时间戳）
- 无需数据库服务器，轻量即可

---

## 关键风险

### lazymanchat.com 接口对接

- 该网站没有公开API文档，需要通过 Chrome 开发者工具抓取网络请求分析内部API格式
- 已确认需要登录态，需分析登录接口、登录凭据如何携带、Session过期策略
- 后端需实现自动登录 + Session保活机制
- **建议在正式开发前先完成接口抓取和登录流程验证**

---

## 不做的（YAGNI）

- 不做流式输出（打字机效果），MVP用一次性返回
- 不做用户注册/登录系统
- 不做多设备同步
- 不做消息搜索
